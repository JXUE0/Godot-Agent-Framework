# Security Policy

## Reporting a vulnerability
If you discover a security issue, please report it privately:
- Open a GitHub issue with the label `security` **only if** it does NOT expose sensitive details.
- For sensitive issues, email the maintainer directly (add contact here).

## Scope
- MCP addon template
- Validation scripts
- Documentation and tooling

## Guidelines
- Do not publish exploits or tokens in public issues.
- Provide minimal reproduction steps.
- Include environment details when possible.

## Response
We aim to acknowledge reports within 72 hours and provide a fix timeline.
