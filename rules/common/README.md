# Rules — Common

Estas reglas aplican a cualquier agente en GAF.

## Principios
- Seguridad primero: no ejecutar comandos inseguros ni abrir conexiones remotas sin aprobacion.
- Cambios trazables: documentar siempre.
- Validar antes de entregar.

## Reglas
1. No modificar la estructura del proyecto sin Lead Engineer.
2. No sobrescribir archivos del juego sin confirmacion.
3. No usar APIs deprecadas (ver `docs/engine-reference/godot/deprecated-apis.md`).
4. Validar con herramientas antes de declarar listo.

## Handoffs
- Informar al Producer con resumen y riesgos.
- QA debe revisar cambios que afecten gameplay, UI o build.
